3 Install Vercel CLI

üíª Install Vercel Command Line Tool

Open your Termux terminal and run:

‚ö†Ô∏è Troubleshooting:
  • If you get permission errors, the command should still work in Termux

‚úÖ Verify Installation

Check that Vercel CLI is installed:

You should see a version number like 34.0.0 or similar.

4 Login to Vercel

üîê Authenticate with Vercel

Run the login command:

What happens next:
  1. Vercel will ask: "Log in to Vercel"
  2. Choose your preferred method (email or GitHub)
  3. A browser will open (or you'll get a link to copy)
  4. Confirm the login in your browser
  5. Return to terminal - you'll see "Success!"

5 Prepare Your Project

üìÅ Navigate to Project Directory

Make sure you're in the right folder:

cd /data/data/com.termux/files/home/proj/babyname2

üß™ Test Your Build

Make sure your app builds successfully:

This takes 2-3 minutes. You should see "Compiled successfully!"

If build fails:
  • Run: npm install first
  • Check for any error messages
  • Make sure all dependencies are installed

6 Deploy to Vercel

üöÄ Initial Deployment

Deploy your app to Vercel:

vercel
Vercel will ask you questions:
  1. "Set up and deploy?" ‚Üí Press Y (Yes)
  2. "Which scope?" ‚Üí Press Enter (use your account)
  3. "Link to existing project?" ‚Üí Press N (No, create new)
  4. "What's your project's name?" ‚Üí Type: soulseed or your preferred name
  5. "In which directory?" ‚Üí Press Enter (current directory)
  6. "Want to override settings?" ‚Üí Press N (No)

Wait 2-3 minutes... Vercel will build and deploy your app!

9 Update Firebase & OAuth

üî• Add Domain to Firebase

Update Firebase to allow your new domain:

üîó Open Firebase Console
Steps:
  1. Select your project (babynames-app-9fa2a)
  2. Go to Authentication
  3. Click Settings tab
  4. Scroll to Authorized domains
  5. Click Add domain
  6. Enter: yourdomain.com (your new domain)
  7. Click Add

üîê Update Google OAuth

Add your domain to Google OAuth authorized origins:

üîó Google Cloud Console
Steps:
  1. Select your project
  2. Click on your OAuth 2.0 Client ID
  3. Under Authorized JavaScript origins, click + ADD URI
  4. Add: https://yourdomain.com
  5. Under Authorized redirect URIs, click + ADD URI
  6. Add: https://yourdomain.com/__/auth/handler
  7. Click Save

üîß Common Issues & Solutions

‚ùå Build Failed

Solution:
  • Run npm run build locally first
  • Fix any errors shown
  • Make sure all dependencies are in package.json
  • Commit changes and redeploy

‚ùå Login Not Working

Solution:
  • Check Firebase authorized domains includes your new domain
  • Check Google OAuth redirect URIs are correct
  • Wait 5 minutes for DNS to propagate
  • Clear browser cache and try again

‚ùå Environment Variables Not Working

Solution:
  • Check all 11 variables are added in Vercel dashboard
  • Make sure "All Environments" is selected
  • Redeploy with vercel --prod

‚ùå Domain Not Loading

Solution:
  • Wait 1-2 minutes for DNS propagation
  • Check domain is added in Vercel project settings
  • Try accessing with https:// prefix
  • Clear browser DNS cache

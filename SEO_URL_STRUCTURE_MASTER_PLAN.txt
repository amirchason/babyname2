3. URL STRUCTURE DESIGN
================================================================================

URL PATTERNS:

INDIVIDUAL NAME PAGES:
Format: /names/[name-slug]
Examples:
  âœ… /names/emma
  âœ… /names/olivia
  âœ… /names/muhammad
  âœ… /names/aria-rose (compound names use hyphen)
  âœ… /names/jean-luc
  âŒ /names/Emma (NO - must be lowercase)
  âŒ /names/emma.html (NO - use Vercel rewrites for clean URLs)

CATEGORY INDEX PAGES:
Format: /names/[category-slug]
Examples:
  âœ… /names/girl
  âœ… /names/boy
  âœ… /names/unisex
  âœ… /names/german
  âœ… /names/popular
  âœ… /names/a (letter category)

CATEGORY PAGE WITH PAGINATION:
Format: /names/[category-slug]/page-[number]
Examples:
  âœ… /names/girl/page-1
  âœ… /names/german/page-2
  âœ… /names/popular/page-5

CONFLICT RESOLUTION:
If a name conflicts with a category (e.g., name "Popular"):
- Priority: Category pages take precedence
- Solution: Name page becomes /names/popular-name
- Document all conflicts in conflicts.json

VERCEL REWRITES (vercel.json):
```json
{
  "rewrites": [
    {
      "source": "/names/:name",
      "destination": "/names/:name.html"
    },
    {
      "source": "/names/:category/page-:page",
      "destination": "/names/:category-page-:page.html"
    }
  ]
}
```

CANONICAL URL STRATEGY:
- Static HTML page: <link rel="canonical" href="https://soulseedbaby.com/names/emma">
- React modal: Add canonical tag pointing to static page
- Prevents duplicate content penalties

6. GENERATION SCRIPT SPECIFICATIONS

SCRIPT: scripts/seo-generation/generate.js

USAGE:
node scripts/seo-generation/generate.js [OPTIONS]

OPTIONS:
--dry-run              Validate without writing files
--resume               Resume from last checkpoint
--count=N              Generate N names (default: all)
--sort=popularity      Sort order (popularity, alphabetical, random)
--skip-existing        Don't overwrite existing files
--categories=TYPE      Generate specific category type (gender, origin, letter, etc.)
--names-file=PATH      Generate only names from file
--output=DIR           Output directory (default: public/names)
--checkpoint-freq=N    Save checkpoint every N names (default: 1000)
--validate             Validate generated files
--quiet                Suppress progress output
--verbose              Detailed logging

EXAMPLES:
# Dry-run test
node scripts/seo-generation/generate.js --dry-run --count=10

# Generate top 100 popular names
node scripts/seo-generation/generate.js --count=100 --sort=popularity

# Generate missing priority names
node scripts/seo-generation/generate.js --names-file=missing-popular.txt --skip-existing

# Resume interrupted generation
node scripts/seo-generation/generate.js --resume

# Generate all with custom checkpoint frequency
node scripts/seo-generation/generate.js --checkpoint-freq=5000

# Generate gender category pages only
node scripts/seo-generation/generate.js --categories=gender

# Full generation (all 174K names)
node scripts/seo-generation/generate.js --sort=popularity --skip-existing

SCRIPT STRUCTURE:

```javascript
// scripts/seo-generation/generate.js

const fs = require('fs-extra');
const path = require('path');
const { loadNames } = require('./loadNames');
const { generateNameHTML } = require('./generateNameHTML');
const { generateCategoryHTML } = require('./generateCategoryHTML');
const { saveCheckpoint, loadCheckpoint } = require('./checkpoint');
const { validateHTML } = require('./validate');

// Configuration
const CONFIG = {
  dataDir: 'public/data',
  outputDir: 'public/names',
  stateFile: 'public/.generation-state/state.json',
  checkpointFreq: 1000,
  batchSize: 100
};

// Main execution
async function main(options) {
  console.log('ğŸš€ Starting SoulSeed SEO generation...\n');

  // Load state if resuming
  let state = options.resume ? await loadCheckpoint(CONFIG.stateFile) : null;

  // Load all names
  console.log('ğŸ“¦ Loading name data...');
  const allNames = await loadNames(CONFIG.dataDir);
  console.log(`   Loaded ${allNames.length} names\n`);

  // Sort names
  const sortedNames = sortNames(allNames, options.sort);

  // Determine start index
  const startIndex = state ? state.lastProcessedIndex : 0;
  const endIndex = options.count ? Math.min(startIndex + options.count, sortedNames.length) : sortedNames.length;

  console.log(`ğŸ“ Generating ${endIndex - startIndex} name pages...`);
  console.log(`   Start: ${startIndex}, End: ${endIndex}\n`);

  // Generate name pages
  let generatedCount = 0;
  let errorCount = 0;
  const errors = [];

  for (let i = startIndex; i < endIndex; i++) {
    const name = sortedNames[i];

    try {
      // Check if should skip
      if (options.skipExisting && await fileExists(name)) {
        continue;
      }

      // Generate HTML
      const html = await generateNameHTML(name);

      // Validate if not dry-run
      if (!options.dryRun) {
        if (options.validate) {
          const validation = await validateHTML(html);
          if (!validation.valid) {
            throw new Error(`Validation failed: ${validation.errors.join(', ')}`);
          }
        }

        // Write file
        const filename = `${name.slug}.html`;
        const filepath = path.join(CONFIG.outputDir, filename);
        await fs.writeFile(filepath, html, 'utf8');
      }

      generatedCount++;

      // Progress logging
      if (!options.quiet && generatedCount % 100 === 0) {
        const percent = ((i - startIndex) / (endIndex - startIndex) * 100).toFixed(1);
        console.log(`   Progress: ${generatedCount} / ${endIndex - startIndex} (${percent}%) - ${name.name}`);
      }

    } catch (error) {
      errorCount++;
      errors.push({ name: name.name, error: error.message });
      if (options.verbose) {
        console.error(`   âŒ Error generating ${name.name}: ${error.message}`);
      }
    }

    // Save checkpoint
    if (generatedCount % CONFIG.checkpointFreq === 0) {
      await saveCheckpoint({
        lastProcessedIndex: i + 1,
        totalGenerated: generatedCount,
        errors: errors,
        timestamp: new Date().toISOString()
      });

      if (!options.quiet) {
        console.log(`   ğŸ’¾ Checkpoint saved at index ${i + 1}`);
      }
    }
  }

  // Final checkpoint
  await saveCheckpoint({
    lastProcessedIndex: endIndex,
    totalGenerated: generatedCount,
    errors: errors,
    timestamp: new Date().toISOString(),
    complete: true
  });

  // Summary
  console.log('\nâœ… Generation complete!');
  console.log(`   Total generated: ${generatedCount}`);
  console.log(`   Errors: ${errorCount}`);
  if (errorCount > 0) {
    console.log(`   See errors in checkpoint file`);
  }
}

// CLI argument parsing
const args = process.argv.slice(2);
const options = parseArgs(args);

// Execute
main(options).catch(error => {
  console.error('âŒ Fatal error:', error);
  process.exit(1);
});
```

CHECKPOINT TRIGGERS:
1. Every 1000 names generated
2. On error (immediate save)
3. On SIGINT/SIGTERM (graceful shutdown)
4. Manual trigger (SIGUSR1 signal)

RESUME BEHAVIOR:
When --resume flag is used:
1. Read state.json
2. Load lastProcessedIndex
3. Skip already-generated files
4. Continue from next name
5. Merge errors from previous run

RECOVERY SCENARIOS:

SCENARIO 1: Script Crashes
- State file preserves progress
- Resume with: --resume
- Skips completed files
- Continues from last index

SCENARIO 2: Termux/App Closes
- State file auto-saved
- Resume same command with --resume
- No data loss

SCENARIO 3: Corrupted Output
- Validation detects issues
- Error logged to state file
- Can regenerate specific files
- Use: --names-file=errors.txt

SCENARIO 4: Disk Full
- Generation stops gracefully
- State file saved before crash
- Free up space
- Resume with --resume

MANUAL RECOVERY:
```bash
# View current state
cat public/.generation-state/state.json | jq

# Resume from checkpoint
node scripts/seo-generation/generate.js --resume

# Regenerate failed names
node scripts/seo-generation/generate.js --names-file=errors.txt

# Validate all generated files
node scripts/seo-generation/validate.js --all --fix
```

14. ROLLBACK PROCEDURES
================================================================================

WHEN TO ROLLBACK:
- Critical errors affecting site functionality
- SEO penalties or ranking drops
- Performance degradation (Lighthouse <70)
- User complaints about broken features
- Security vulnerabilities discovered

ROLLBACK STEPS:

OPTION 1: Vercel Instant Rollback
â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”
1. Login to Vercel dashboard
2. Go to project deployments
3. Find previous stable deployment
4. Click "Promote to Production"
5. Confirm rollback

Time to rollback: <1 minute

OPTION 2: Git Revert + Redeploy
â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”
```bash
# Revert to previous commit
git log --oneline (find commit hash)
git revert [commit-hash]
git push origin master

# Redeploy
npm run deploy
```

Time to rollback: 5-10 minutes

OPTION 3: Restore from Backup
â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”
```bash

cd /path/to/project
tar -xzf /path/to/backup.tar.gz

# Redeploy
npm run deploy
```

Time to rollback: 10-15 minutes

PARTIAL ROLLBACK:
If only specific pages are problematic:
1. Identify problematic pages
2. Remove those files from public/names/
3. Regenerate using generation script
4. Redeploy

POST-ROLLBACK:
[ ] Verify site functionality restored
[ ] Communicate to users (if public issue)
[ ] Investigate root cause
[ ] Fix issues in development
[ ] Test thoroughly before re-deploying

APPENDIX A: URL CONFLICT RESOLUTION
================================================================================

POTENTIAL CONFLICTS:

Name "Popular" conflicts with category /names/popular
Name "Girl" conflicts with category /names/girl
Name "Boy" conflicts with category /names/boy

RESOLUTION STRATEGY:
1. Categories take precedence (higher SEO value)
2. Conflicting names get suffix: /names/popular-name
3. Document all conflicts in url-conflicts.json
4. Add redirects if needed

CONFLICTS FILE: scripts/seo-generation/url-conflicts.json
```json
{
  "conflicts": [
    {
      "name": "Popular",
      "conflictsWith": "/names/popular (category)",
      "resolution": "/names/popular-name",
      "status": "resolved"
    }
  ]
}
```

APPENDIX C: TROUBLESHOOTING
================================================================================

ISSUE: Generation script fails with "out of memory"
SOLUTION: Increase Node.js memory limit
```bash
NODE_OPTIONS='--max-old-space-size=4096' node scripts/seo-generation/generate.js
```

ISSUE: Checkpoint resume not working
SOLUTION: Check state file integrity
```bash
cat public/.generation-state/state.json | jq
# If corrupted, manually edit or delete and restart
```

ISSUE: HTML validation errors
SOLUTION: Check template syntax
```bash
node scripts/seo-generation/validate.js --file=public/names/test.html --verbose
```

ISSUE: Vercel deployment timeout
SOLUTION: Deploy in smaller batches or increase timeout in vercel.json

ISSUE: Sitemap too large
SOLUTION: Already split into 50K URL chunks (max per XML file)

ISSUE: React routing conflicts with static pages
SOLUTION: Check vercel.json rewrites configuration

ISSUE: Duplicate content penalty
SOLUTION: Verify canonical tags present on all modal views
